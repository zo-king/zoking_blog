# P21 关于页与友链页设计

日期：2026-07-13

## 关于页

`apps/site/layouts/single.html` 只在 `.Section == "post"` 时渲染相关文章 partial。关于、友链和其他独立页面不再自动出现推荐文章，因此页面只展示自身正文、版权和必要的页面结构。

## 友链页

友链继续由页面 front matter 的 `links` 数组驱动，不新增数据库表。每项显示：

- 站点 favicon/头像；
- 站点名称；
- 人工维护的简介；
- 规范化 host；
- 外链箭头和新窗口行为。

页面使用两列重复卡片，移动端变为单列。头像使用三段式策略：

1. 直接请求站点自身 `https://host/favicon.ico`；
2. 失败后请求 `https://icons.duckduckgo.com/ip3/host.ico`；
3. 两者都失败时显示标题首字母。

图片在成功前透明，失败时从 DOM 移除，避免浏览器破图图标进入视觉界面。请求使用 `referrerpolicy="no-referrer"`，不携带账号或 Token；第三方 favicon 服务属于外部请求，应在生产隐私说明中记录。

后续友链数量增长到 20 条以上时，再评估构建期下载头像并存入媒体库，避免每次阅读都依赖第三方站点；构建期抓取必须有超时、大小限制、域名校验和失败回退。
