# Admin 视觉与交互规范

本文是 Zoking Blog B 端后台的实现级视觉规范。组件能力来自 Arco Design，页面结构与品牌表达由本项目控制。

## 1. 设计目标

- 安静、可靠、可连续工作，不做营销页表达。
- 中文信息优先，表格和表单保持中高密度。
- 内容、运营、系统三类任务有明确导航归属。
- 桌面端高效扫描，390px 窄屏仍可完成关键操作。
- 状态、危险操作和反馈含义一致。

## 2. 视觉 Token

| Token | 值 | 用途 |
|---|---|---|
| Canvas | `#f4f6f5` | 页面底色 |
| Surface | `#ffffff` | 工具面板、表格、表单 |
| Ink | `#1d2824` | 主文字 |
| Muted | `#66736e` | 辅助文字 |
| Line | `#dfe6e2` | 普通边框 |
| Primary | `#147d68` | 主操作、当前导航、焦点 |
| Coral | `#cf654a` | 品牌辅助和需要注意的信息 |
| Sidebar | `#19231f` | 桌面主导航 |

约束：

- 卡片和工具面板圆角不超过 6px。
- 不使用渐变、装饰光斑、悬浮页面分区或卡片嵌套卡片。
- 不使用负字距，不根据视口宽度缩放字号。
- 阴影只允许用于弹窗、下拉菜单等真正浮层。

## 3. 尺寸与密度

- 桌面侧栏：236px。
- 全局页头：64px。
- 页面最大内容宽度：1680px，居中。
- 桌面页面边距：30px；窄屏：12px 至 20px。
- 标准组件高度：32px；登录输入和主按钮：48px。
- 页面标题：24px；面板标题：15px；正文和表格：13px 至 14px。
- 页面面板间距：18px；表单项间距：18px。

## 4. 导航

- 桌面端使用固定可见的深色分组侧栏。
- 当前项使用浅绿色底和深绿色文字，不只依赖颜色，文字同时加粗。
- 移动端侧栏完全隐藏，由页头菜单按钮打开左侧 Drawer。
- 导航点击必须更新 URL，浏览器前进、后退和刷新后选中状态保持正确。
- 无权限菜单不渲染，直接访问无权限路由显示中文警告且不挂载业务页。

## 5. 页面结构

标准顺序：

```text
全局页头
  -> 页面标题、说明、主操作
  -> 指标或状态摘要（确有需要时）
  -> 一个或多个内容工具面板
  -> 表格、表单或详情工具
```

- 页面标题区不放入卡片。
- `ContentPanel` 只用于真正需要边界的表格、编辑器和管理工具。
- 同类工具可以并排，但在窄屏必须自动堆叠。
- 页面主操作放在页面标题右侧；面板局部操作放在面板标题右侧。

## 6. 表格

- 表头使用浅灰绿色背景、12px 半粗字体。
- 行高应便于扫描，悬停只做轻微底色反馈。
- 状态用中文 Tag，不能直接展示内部英文枚举。
- 常用行操作使用图标加短文本；删除使用危险色和 Popconfirm。
- 空列表显示面向任务的中文说明。
- 长内容截断或换行，表格配置最小横向宽度，禁止列互相覆盖。

## 7. 表单与编辑

- 标签置于控件上方，必填规则由 Arco Form 管理。
- 字段按“内容、发布、SEO、系统”组织，不连续堆叠无关字段。
- Boolean 使用 Switch，枚举使用 Select，数字使用 InputNumber。
- 保存为普通或次主按钮，发布为 Primary，删除为 Danger。
- 异步操作必须显示 loading 并防止重复提交。
- Markdown 正文编辑区必须保持稳定高度，窄屏不横向溢出。

## 8. 状态与反馈

- 正常、成功、已发布：绿色。
- 处理中、排队、请求中：蓝色。
- 草稿、待审核、需注意：橙色或金色。
- 失败、拒绝、删除：红色。
- 停用、取消、未知：灰色。
- 页面级错误使用 Alert；短时操作结果使用 Message；高危确认使用 Popconfirm 或 Modal。

## 9. 响应式与无障碍

- 960px 以下切换移动导航。
- 760px 以下页面标题和面板标题操作区换行。
- 390px 验收时页面根节点不得出现横向滚动。
- 图标按钮必须提供 `aria-label` 和 Tooltip。
- 键盘焦点必须可见；Modal 和 Drawer 保持焦点锁定。
- 颜色不是状态的唯一表达，Tag 同时提供中文文本或图标。

## 10. 禁止项

- 不混用多个完整组件库。
- 不在 JSX 中大量写一次性内联布局样式。
- 不显示英文后台枚举作为最终文案。
- 不把列表、编辑器、设置、发布等所有领域重新堆到一个路由。
- 不用大面积统计卡片填充空白。
- 不用说明性文案向用户解释 UI 本身如何使用。
