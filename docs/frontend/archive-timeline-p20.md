# P20 搜索降级与文章时间线设计

日期：2026-07-13

## 1. 信息架构

- 主导航：保留首页、关于、归档、项目、友链；移除搜索。
- 首页右栏：只保留标签云，搜索不再与主导航重复。
- 搜索能力：保留 `/search/`、`/search/index.json`、搜索实现和 404 搜索，不删除未来可恢复能力。
- 归档：顶部分类/标签负责主题探索，时间线负责按发布时间回看，三者职责不同。

## 2. 模板结构

`apps/site/layouts/archives.html` 显式执行 `.ByDate.Reverse`，再按发布年份分组，防止未来全局排序改成 `lastmod` 后破坏时间线顺序。每个年份使用 `section`，条目使用 `ol > li > article`，标题层级固定为 `H1 归档 -> H2 年份 -> H3 文章`。

`apps/site/layouts/_partials/article-list/timeline.html` 独立于上游 compact partial，避免上游固定 H2 造成层级错误。每条包含 RFC3339 `time[datetime]`、文章标题、描述和可选 56px 缩略图；缩略图不承担唯一语义并退出辅助技术树。

## 3. 视觉与响应式

- 桌面：88px 年份轨道，74px 日期轨道，1px 连接线和 9px 节点；文章行无额外卡片和阴影。
- 390/320px：年份回到内容上方，日期回到标题上方，轴线保持左侧 26px 缩进。
- 缩略图桌面/普通移动为 56px，360px 以下为 48px；标题允许任意位置换行。
- 链接最小高度 44px，键盘焦点使用主题 accent outline。
- 悬停只改变标题颜色和图片透明度；`prefers-reduced-motion: reduce` 禁用过渡。
- 明暗主题只使用 Stack token，不引入单独硬编码主题色。

## 4. 扩展边界

- 文章达到 20 至 30 篇后再评估是否增加单一图标搜索入口。
- 单年接近 50 篇后可加入月份标题和数量，但不能增加客户端滚动监听。
- 内容成熟度、最后维护时间必须先有真实后台字段和发布契约，不能通过发布日期推断。
- 上游 Theme Stack 升级时只需复核本地 archive template、timeline partial 和独立 SCSS，不直接修改上游核心。
