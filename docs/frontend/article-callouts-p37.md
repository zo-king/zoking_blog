# P37 技术文章提示块

## 用法

文章 Markdown 使用 Hugo/Theme Stack 已支持的 alert 语法：

```markdown
> [!NOTE]
> 这里是背景说明。

> [!TIP]
> 这里是实践建议。

> [!IMPORTANT]
> 这里是必须遵守的工程不变量。

> [!WARNING]
> 这里是风险提示。
```

当前已在 Gin 生产加固、Go Context 与并发、GORM 事务与锁三篇真实技术文章中使用。

## 视觉约束

- 沿用 Theme Stack 的 `alert-*` 颜色语义，不新增品牌色。
- 使用轻微圆角和一级阴影，与文章内代码块保持同一层级。
- 标题、图标和正文有明确间距，中文长句允许自然换行。
- 700px 以下缩小内边距；320px 不得横向溢出。
- `prefers-reduced-motion` 下没有额外动效。
- 打印模式取消阴影并保留边框，使提示内容不因打印丢失。

## 非目标

不把提示块改成可折叠面板，不添加脚本、不做悬浮弹窗、不把普通引用全部自动改成 alert，也不在首页增加新的组件区。
