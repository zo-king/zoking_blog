# 前后台前端规划

## 1. C 端前台设计要求

C 端必须保留 Hugo Theme Stack 的整体观感：

- 左侧站点栏。
- 中间内容区。
- 右侧信息栏。
- 卡片式文章列表。
- 文章详情页阅读体验。
- 明暗色模式。
- 移动端折叠菜单。
- 代码块、引用、表格、图片、标题层级完整样式。

原则：

- 不把前台改造成营销落地页。
- 不破坏 Stack 的阅读密度和布局节奏。
- 优先复用 Hugo/Stack 模板。
- 动态能力以局部 JS 增强形式接入。

## 2. B 端后台设计要求

后台是生产工具，不做营销型视觉。

当前技术选型：

- `apps/admin` 当前采用 React 18.3.1 + Vite + Arco Design React 2.66.15。
- 路由固定使用 react-router-dom 7.18.1。
- 前端依赖安全审计基线为 npm audit 0 漏洞。

设计要求：

- 左侧主导航。
- 顶部显示当前用户、站点入口、通知、快捷发布。
- 列表页统一支持搜索、筛选、排序、批量操作。
- 表单页统一支持保存草稿、预览、提交审核、发布。
- 高危操作必须二次确认。
- 所有空状态、错误状态、加载状态完整。
- 后台信息密度适合反复操作。

  - 页面管理。
  - 分类管理。
  - 标签管理。

- 评论审核。
- 发布中心。
- 用户权限。

  - 角色管理。
  - 权限管理。

  - 主题设置。
  - SEO 设置。
  - 导航菜单。

## 4. 页面级要求

数据看板：

- 指标卡片。
- 趋势图。
- 热门文章。
- 最近发布。
- 待处理评论。
- 最近操作。

文章管理：

- 表格列表。
- 状态筛选。
- 分类/标签筛选。
- 关键词搜索。
- 批量操作。
- Markdown 编辑器。
- C 端主题预览。
- 发布操作。

媒体库：

- 网格视图。
- 列表视图。
- 上传。
- 复制链接。
- 引用关系。
- 删除保护。

评论审核：

- 评论内容。
- 关联文章。
- 提交人信息。
- IP/User Agent。
- 审核操作。
- 黑名单操作。

发布中心：

- 发布任务列表。
- 当前状态。
- 构建日志。
- 失败原因。
- 重试。
- 回滚。

## 5. 前端验收标准

C 端：

- 首页、详情页、分类、标签、归档、搜索可用。
- 桌面和移动端均可读。
- Stack 视觉效果保留。
- 发布内容正确反映。
- 评论、阅读量、点赞等动态能力失败时不影响阅读。

B 端：

- 管理员能完成“写文章 -> 上传封面 -> 选分类标签 -> 预览 -> 发布 -> 前台查看”闭环。
- 权限边界正确。
- 表单校验明确。
- 加载、空状态、错误状态齐全。
- 高危操作有确认。
