# P36 文章年份归档前端设计

## 页面职责

`/post/` 是全部公开文章的扫描入口，`/archives/` 是个人获奖与成果时间线。两者分别面向内容阅读和个人履历，不共享数据源，也不复用相同标题。

## 模板

`apps/site/layouts/post/list.html` 覆盖 Theme Stack 的 post section 列表模板。页面直接读取 `Site.RegularPages` 中 `Section == post` 的内容，按发布日期倒序并以年份分组。

语义结构固定为：

- 唯一 `h1`：文章。
- `nav[aria-label="文章年份"]`：年份锚点与篇数。
- 每个年份为带 `aria-labelledby` 的 `section`。
- 年份标题为 `h2`，文章标题为 `h3`。
- 每年文章使用 `ol > li > article`。
- 发布日期使用机器可读 `time[datetime]`。

## 视觉与响应式

页面保持无卡片、无阴影的 Stack 紧凑列表。桌面条目为日期、标题、元信息三列；700px 以下元信息进入标题下方；360px 以下日期、标题和元信息完全单列。标题链接最小高度 44px，长中文和英文单词允许自然换行。

年份定位只使用 `:target` 和 CSS keyframes。动画不拦截锚点、不改变历史记录；reduced-motion 环境不执行动画。页面不加载运行时 JavaScript。

## 数据与隐私

页面仅使用构建时已有文章元数据。不存在访客侧 API、远程头像、统计、Cookie 或本地存储。分类为空时省略分类链接；阅读时长遵循文章和站点现有参数。

## 扩展边界

文章数量达到数百篇后，可让旧年份使用原生 `details` 或增加构建时月份分组，但不得依赖 JavaScript 才能访问旧文章。独立随机页面、脚注边注和第三方内容保护壳分别作为后续任务，不混入 P36。
