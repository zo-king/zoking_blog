# 系列管理与阅读体验

日期：2026-07-13
任务：`SERIES-P18-001`

## 1. Admin 信息架构

系列归入现有“内容组织”路由，和分类、标签并列为第三个 Tab，不新增顶级菜单或长页面。Tab 内容采用紧凑表格；创建和编辑使用 Modal，删除使用二次确认。

系列表单字段：名称、Slug、说明、封面、展示顺序、启用状态。封面复用媒体库 ID，不接受任意外部 URL。API 返回被引用冲突时保留 Modal/表格状态，并给出“请先调整相关文章”的可操作提示。

## 2. 文章编辑器

文章右侧“内容组织”增加：

- 单选系列，可清除。
- 系列序号，选择系列后启用，只接受 `>=1` 的整数。

未选系列时序号清空并禁用；已选系列时序号必填。新建、选择已有文章、保存响应、Preview 和 Publish 响应都必须恢复这两个字段，不能因刷新 taxonomy 数据而丢失。

系列选择不改变分类和标签。质量检查、保存、预览、发布仍遵循现有顺序，前端校验只提供即时反馈，服务端约束是最终真相。

## 3. C 端系列模块

属于系列的文章显示独立“所属系列”模块：

- 展示系列名和当前位置。
- 目录按 `series.order` 排序，当前项高亮并含屏幕阅读器可读的“正在阅读”。
- 提供系列内上一篇/下一篇。
- 保留现有全站上一篇/下一篇、阅读进度和分享区。

目录从 Hugo 构建期的 `.Site.RegularPages` 计算，不在浏览器运行时调用 API。这样 release 是可缓存、自包含的静态产物，API 暂时不可用也不影响阅读。

## 4. 布局与可访问性

- 延续 Stack 的颜色、边框、阴影和字号 token，不做营销式大卡片。
- 模块不嵌套卡片；目录过长时仅目录区域受控滚动，页面主滚动保持自然。
- 1280x720 下不把文章主体挤出可读宽度；390x844 下链接、序号和长标题可换行，无横向滚动。
- 链接具有键盘焦点；当前项不能只靠颜色；上一篇/下一篇文案包含方向和文章标题。
- 深色模式保持正文、次要文字、边框和焦点对比度。

## 5. 空态与异常数据

- 无 series front matter：完全不渲染系列模块。
- 系列只有当前文章：显示目录和当前位置，不渲染空的前后导航。
- 同一 release 中未找到其他文章：按单篇系列处理。
- 构建器已经拒绝缺 slug/name/order 的快照；模板仍对缺字段做条件保护，不能让 Hugo build panic。
- 普通文章导航不依赖系列模块是否成功渲染。
