# P20 个人博客归档与时间线调研

调研日期：2026-07-13。目标是解决当前 3 篇文章下搜索入口偏重的问题，并寻找能融入 Hugo Theme Stack 的低干扰时间表达。所有模式只提炼信息结构，不复制站点品牌视觉。

## 1. 真实站点样例

| 站点 | 页面 | 值得借鉴 | 当前不采用 |
|---|---|---|---|
| Anthony Fu | [Posts](https://antfu.me/posts) | 年份作为清晰视觉锚点，文章行紧凑，悬停反馈很轻 | 巨型年份水印和植物线稿会抢占当前 Stack 三栏版面 |
| Simon Willison | [2026 archive](https://simonwillison.net/2026/) | 年/月分组、数量统计和年份定位适合高频发布 | 当前只有 3 篇且集中在一个月，多级目录会产生空洞层级 |
| Maggie Appleton | [Garden](https://maggieappleton.com/garden) | 内容成熟度和持续维护时间能表达数字花园生命周期 | 本站尚无“成长中/常青”等真实业务字段，不能由日期臆测状态 |
| Derek Sivers | [Articles](https://sive.rs/blog) | 搜索降为文章集合内的低频工具，日期列表克制直接 | 暖色纸张和复古字体不属于现有 Stack 视觉语言 |
| Gwern | [Blog Posts](https://gwern.net/blog/index) | 反向时间索引、年份分隔和紧凑日期效率高 | 边注、固定工具栏和高密度项目符号对当前站点过重 |

## 2. 采用结论

- 一级导航不再显示搜索，首页右栏不再显示搜索组件。
- `/search/` 和 JSON 索引继续存在，供旧链接、直接访问和 404 恢复使用。
- `/archives/` 继续作为统一内容发现中心，分类与标签保留在顶部，但不回到一级导航。
- 文章使用“年份 + 数量 + 竖线节点 + 月日 + 标题/摘要 + 可选小图”的单列时间河。
- 当前不增加月份层、活动热力图、左右交替轴、滚动监听、淡入动画、粒子或无意义插画。
- 文章超过 20 至 30 篇时重新评估显式搜索入口；单年文章接近 50 篇时再评估月份分组；总量接近 1000 篇时再评估年度分页。

## 3. 设计原则

装饰必须承载时间或内容信息。年份、文章数量、节点、连接线和封面都能帮助扫描；纯背景光效、视差和持续动画不能。颜色只复用 `--accent-color`、`--card-separator-color` 和主题文本 token，避免形成一套与 Stack 冲突的视觉系统。
