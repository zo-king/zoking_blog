# 个人网站与博客特色功能调研

调研日期：2026-07-13
任务：`SITE-AUDIT-P16-001`

## 1. 调研目标

本轮不以堆功能为目标，而是筛选同时满足以下条件的能力：

- 对 C 端阅读者有明确价值；
- 保留 Hugo Theme Stack 的静态阅读体验；
- 能与现有 Gin/GORM/PostgreSQL/Admin 发布链路自然衔接；
- 隐私、安全和运维成本可解释、可测试；
- 不把统计、账号或第三方服务伪装成“低成本小功能”。

## 2. 真实站点与可借鉴模式

| 来源 | 实际模式 | 可借鉴点 | 当前判断 |
|---|---|---|---|
| [Simon Willison's Weblog](https://simonwillison.net/) | 首页把 Entries、Links、Quotes、Notes、Guides 等内容流并列组织 | 不同内容类型可以共享时间流，但需要清晰的信息类型和筛选 | 适合后续“笔记/链接收藏”，不必马上新增数据库模型 |

| [Derek Sivers /now](https://sive.rs/now) | 单独维护“我现在在做什么”页面并显示更新时间 | 低成本、高个人辨识度，不需要新系统 | 可直接复用现有独立页面编辑/发布能力 |

评分：价值、契合度越高越好；成本、风险越低越好。每项 1-5。

| 搜索失败恢复与标题权重 | 4 | 5 | 1 | 1 | P16 已实现 |
| Now / Uses / Projects 页面 | 4 | 5 | 1 | 1 | 建议直接用现有页面系统运营 |

## 4. 本轮实现

- 文章顶部阅读进度条，使用可访问的 `progressbar` 语义。
- 阅读到 5%-98% 时按文章路径保存进度，500ms 节流写入；完成后删除，30 天后自动过期。
- 重访文章显示“继续阅读（N%）”，尊重 `prefers-reduced-motion`。
- 搜索标题命中权重提升为正文命中的 5 倍。
- 搜索索引失败显示中文错误和原地重试；响应状态和 JSON 结构均校验。
- 搜索结果补封面 `alt`、H1/H2/H3 层级和动态播报。
- 移动菜单支持 Escape、外部点击关闭和焦点归还。
- 评论网络错误本地化，并通过 live region 播报。
- 深色次要文字对比度提升到 WCAG AA。
- `/post/`、`/page/` section 中文化，左右辅助区域具有唯一名称。
- 生产 Nginx 未知路径返回真实 404 页面，不再 soft 404；文本资源启用 gzip。

## 5. 审计后仍需处理的工程项

### P1：新增高风险领域前必须解决

- API 契约文档列出的刷新/登出、审核、修订、浏览/点赞等能力与实际 Router 存在漂移；新增功能前先确定代码生成或契约校验机制。
- 评论写请求目前统一映射到 `comment:moderate`，发布取消等动作使用较宽的 `publish:create`；角色需要更细粒度时，应拆权限码并补 seed/路由矩阵测试。
- Hugo 允许不安全 HTML，Admin token 存在 `localStorage`；引入投稿、第三方编辑器或可执行嵌入前必须先做内容清洗和 XSS 威胁建模。
- 静态产物 URL 不能提供真正的私密访问控制，会员/私密文章不能沿用当前 public release 方案。

### P2：推荐的产品演进顺序

1. 发布前质量检查：空正文、标题/描述、图片 alt、站内断链和保留 slug。
2. 结构化系列与系列内导航：使用独立关系表维护顺序，不用 JSON 设置或标签冒充长期数据模型。
3. 评论线程：限制深度，父评论必须已审核且属于同一文章。
4. 本地收藏清单：设备内可查看、导出和一键清除，不创建匿名设备 ID。
5. 定时发布：独立设计时区、内容快照、改期/取消和 Worker 竞态。

## 6. 非目标

- 不为了“有数字”立即采集 PV/UV、IP 或设备指纹。
- 不把 Preview 随机链接当作敏感稿件权限系统。
- 不把文章关系塞入 `site_settings` JSON。
- 不在普通文章编辑器默认引入任意 JavaScript；交互内容应使用受控组件或隔离沙箱。
