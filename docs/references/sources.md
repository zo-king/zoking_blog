# 参考资料

本文记录当前工程规划参考过的公开资料。后续若新增技术决策，应继续追加来源。

## Hugo 与 Stack

- [Hugo Theme Stack Getting Started](https://stack.cai.im/guide/getting-started)：用于确认 Stack 推荐通过 starter template/Hugo Modules 起步，并具备 SCSS、TypeScript、GitHub Pages 构建流程。
- [Hugo Theme Stack Comments](https://stack.cai.im/config/comments)：用于确认 Stack 支持多种评论 provider，评论可作为 C 端可插拔增强能力。
- [CaiJimmy/hugo-theme-stack](https://github.com/CaiJimmy/hugo-theme-stack)：用于确认主题定位为 card-style Hugo theme designed for bloggers。
- [Hugo Front Matter](https://gohugo.io/content-management/front-matter/)：用于确认 Hugo front matter 用于描述内容、控制发布时间、draft、title、weight 等内容元数据。
- [Hugo Content Management](https://gohugo.io/content-management/)：用于确认 Hugo 支持 content types、menus、summaries、taxonomies、URL management、page bundles 等内容管理能力。
- [Hugo Taxonomies](https://gohugo.io/content-management/taxonomies/)：用于确认 Hugo taxonomy 可表达分类、标签等内容分组关系。

## CMS 与博客系统

- [Contentful: Headless CMS](https://www.contentful.com/headless-cms/)：用于确认 headless CMS 的核心思想是内容管理层与展示层分离。
- [Strapi: Headless CMS Features](https://strapi.io/blog/headless-cms-features-content-team-productivit)：用于参考 CMS 常见能力，包括编辑体验、角色工作流、协作、预览、版本和权限。
