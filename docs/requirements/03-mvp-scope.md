# MVP 范围与验收

本文件定义博客第一版的需求边界、分期计划和验收标准。它是后续中心窗口给子 agent 分派任务时的主要依据。

## 优先级定义

- P0：第一版必须完成，不完成不应进入实现验收。
- P1：第一版建议完成，若存在外部依赖可延后。
- P2：后续增强，不阻塞第一版。
- Out：明确不在当前阶段。

## P0 必须完成

| ID | 需求 | 验收标准 |
|---|---|---|
| P0-001 | 项目可本地启动 | `hugo server -D --gc --disableFastRender` 可启动，核心页面无构建错误 |
| P0-002 | 生产构建通过 | `hugo --gc --minify` 通过 |
| P0-003 | Stack 主题正确集成 | 首页、文章页、列表页使用 Stack 样式渲染 |
| P0-004 | 首页文章列表 | 首页展示文章列表、分页正常 |
| P0-005 | 文章详情页 | 文章标题、日期、正文、代码块、图片、目录正常 |
| P0-006 | 关于页 | `/about/` 可访问，评论关闭 |
| P0-007 | 归档页 | `/archives/` 可访问，按年份展示文章 |
| P0-008 | 搜索页 | `/search/` 可访问，搜索 JSON 输出存在 |
| P0-009 | 分类和标签 | 分类、标签能从文章 frontmatter 渲染并可访问 |
| P0-010 | RSS | RSS 输出可访问，内容策略明确 |
| P0-011 | SEO 基础 | title、description、canonical、sitemap、OG 基础可用 |
| P0-012 | 移动端可读 | 手机宽度下导航、文章、代码块、图片不明显错位 |
| P0-013 | 明暗色模式 | 默认 auto，可手动切换，刷新后正常 |
| P0-014 | Frontmatter 规范 | 文章模板含 title、description、date、slug、categories、tags、draft |
| P0-015 | Page bundle 规范 | 示例文章使用 bundle，图片放在文章目录 |
| P0-016 | 工作日志 | `docs/process/worklog.md` 存在并记录关键步骤 |
| P0-017 | 上下文接力 | `docs/process/context-handoff.md` 存在，新窗口能接手 |

## P1 应该完成

| ID | 需求 | 验收标准 |
|---|---|---|
| P1-001 | 评论方案 | 明确启用或暂缓；若启用，provider 和隐私策略明确 |
| P1-002 | 统计方案 | 明确启用或暂缓；若启用，cookie 和隐私策略明确 |
| P1-003 | Open Graph 图片 | 重要文章有封面图，分享预览可用 |
| P1-004 | 图片压缩规范 | 文档写明封面和正文图片大小建议 |
| P1-005 | 友情链接或项目页 | 如果用户需要，创建对应页面 |
| P1-006 | 部署 runbook | 部署平台确定后写运行手册 |
| P1-007 | CI 构建 | 平台确定后自动执行 `hugo --gc --minify` |

## P2 后续增强

| ID | 需求 | 说明 |
|---|---|---|
| P2-001 | 多语言 | 需要用户确认中英双语计划 |
| P2-002 | Newsletter | 需要邮件服务和隐私方案 |
| P2-003 | CMS 后台 | 技术博客初期不需要 |
| P2-004 | Webmention / reaction | 需要额外服务 |
| P2-005 | PWA | 静态博客可做，但缓存策略后置 |
| P2-006 | 自动化链接检查 | 内容量上来后再加 |
| P2-007 | Lighthouse / 可访问性自动检查 | 部署流程稳定后加入 |

## Out：当前不做

- 多作者权限系统。
- 动态后台。
- 用户登录系统。
- 服务端搜索。
- 电商、会员、付费墙。
- 大型前端框架重写。

## 分期计划

### Phase 0：需求与工程基线

产物：

- 需求总览。
- 常见博客功能调研。
- Stack 能力映射。
- MVP 范围。
- 子 agent 协作规则。
- 工作日志。
- 上下文接力规则。

验收：

- 文档存在。
- 工作日志记录本阶段。
- 没有开始无授权实现。

### Phase 1：站点骨架

产物：

- Hugo 站点结构。
- 基础配置。
- 必备页面。
- 示例文章。
- 本地预览通过。

验收：

- P0-001 到 P0-010 通过。

### Phase 2：阅读体验

产物：

- 文章模板。
- 图片规范。
- 明暗色检查。
- 移动端检查。
- SEO 基础。

验收：

- P0-011 到 P0-015 通过。

### Phase 3：部署与运维

产物：

- 部署配置。
- 部署 runbook。
- CI 构建。
- 线上验证。

验收：

- P0-016、P0-017 和相关 P1 通过。
