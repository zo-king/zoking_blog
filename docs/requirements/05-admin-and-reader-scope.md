# C 端与 B 端范围规划

## MVP

C 端：

- 首页。
- 文章详情。
- 分类。
- 标签。
- 归档。
- 搜索。
- 评论展示。
- RSS / Sitemap。
- Hugo Theme Stack 视觉效果保留。

B 端：

- 登录。
- 数据看板基础版。
- 文章管理。
- 页面管理。
- 分类标签。
- 媒体上传。
- 评论审核。
- 发布中心基础版。
- 站点基础设置。

后端：

- Gin API。
- GORM 模型。
- PostgreSQL。
- JWT/Session 鉴权。
- RBAC 基础版。
- 审计日志基础版。

发布：

- 草稿。
- 发布。
- 下线。
- 定时发布时间字段。
- 发布任务基础状态。

## P1

- 文章版本历史与回滚。
- 发布中心增强：发布前检查、失败重试、发布记录。
- 媒体库增强：缩略图、压缩、对象存储。
- 评论增强：敏感词、黑名单、邮件通知。
- 数据看板增强：PV/UV、热门文章、趋势图。
- 工作流增强：作者提交、编辑审核、驳回原因。
- 用户权限增强：细粒度权限和操作范围。
- 文章预览链接。
- 发布产物回滚。

## P2

- 多站点管理。
- 多语言内容。
- 前台会员中心。
- 内容导入导出。
- 全文搜索引擎接入。
- Webhook / 插件机制。
- 多环境发布。
- A/B 测试或推荐系统。
- 更完整的数据分析体系。

## C 端验收

- 首页、详情页、分类、标签、归档、搜索在桌面和移动端均可正常使用。
- C 端视觉与 Hugo Theme Stack 的布局、密度、阅读体验保持一致。
- 文章 Markdown 渲染完整，代码块、图片、表格、目录正常。
- 文章发布后可被前台访问。
- SEO 信息正确输出。
- 评论提交、展示、审核状态符合预期。
- 页面加载无明显布局错位。

## B 端验收

- 管理员可完成“写文章、上传封面、设置分类标签、预览、发布、前台查看”的闭环。
- 作者、编辑、管理员权限边界正确。
- 未授权用户不能访问后台接口。
- 删除、发布、下线、修改权限等操作有审计日志。
- 媒体引用关系可追踪。
- 站点设置修改后能反映到 C 端。
- 发布中心能展示待处理内容和发布结果。

## 后端验收

- API 返回结构统一。
- 关键接口有权限校验。
- 数据库迁移可重复执行。
- 文章 slug 唯一且可读。
- 定时发布任务可被测试验证。
- 重要业务有单元测试或集成测试覆盖。
- 错误日志足以定位发布、上传、权限问题。
