# 安全基线

本文定义全栈博客系统的最低安全要求。

## 1. 认证安全

- 密码使用 bcrypt 或 argon2id。
- Access token 短期有效。
- Refresh token 只存 hash。
- 登录失败限流。
- 支持禁用用户。
- 超级管理员不能被删除到 0 个。

## 2. 授权安全

- 后台所有接口默认需要认证。
- 使用 RBAC 权限码。
- 资源级权限在 service 层校验。
- 高危操作必须审计。
- Public API 不暴露后台字段。

## 3. 输入校验

- DTO 层校验类型、长度、枚举、必填。
- slug 只允许安全字符。
- Markdown 渲染必须清洗危险 HTML。
- 评论内容必须过滤脚本。
- 所有 SQL 参数化。

## 4. 上传安全

- 限制文件大小。
- 限制扩展名。
- 校验 MIME 和文件头。
- 随机化存储文件名。
- 防路径穿越。
- SVG 上传需额外清洗或默认禁止。
- 删除媒体前检查引用关系。

## 5. Web 安全

- 生产启用 HTTPS。
- CORS 白名单。
- Cookie 设置 `HttpOnly`、`Secure`、`SameSite`，如使用 Cookie。
- 后台接口限流。
- 管理后台不允许被搜索引擎索引。
- 错误响应不暴露内部路径和堆栈。

## 6. 日志脱敏

不得记录：

- 密码。
- token。
- secret。
- 完整 Cookie。
- 数据库连接串。
- 完整 IP，如无必要。

必须记录：

- request id。
- actor id。
- action。
- resource。
- result。
- error code。

## 7. 数据库安全

- 应用账号不是 superuser。
- 生产数据库不开放公网或限制来源。
- 备份文件加密存储。
- migration 权限与运行期权限可分离。

## 8. 评论反垃圾

MVP：

- 频率限制。
- 待审核。
- IP/email hash。
- 关键词黑名单。

增强：

- 验证码。
- Akismet 或类似服务。
- 用户信誉。

## 9. 安全 DoD

任务完成前检查：

- 新接口是否有鉴权。
- 新写入是否有校验。
- 新高危操作是否有审计。
- 新上传路径是否有限制。
- 新配置是否进入 `.env.example` 且无真实密钥。
- 新日志是否脱敏。
# 审计日志安全边界

- 仅审计后台认证后的写请求，不审计 C 端普通读取。
- IP 使用服务端密钥执行 HMAC-SHA256，不保存明文 IP。
- 不读取或持久化原始请求体，避免密码、token、正文和上传文件进入日志。
- User-Agent 最长保留 512 字符。
- 审计查询要求数据库权限 `audit:read`，权限撤销后对现有 JWT 立即生效。

# RBAC 安全边界

- JWT 不携带角色和权限，撤销角色或权限后立即生效。
- 每个后台请求检查用户 `status=active` 且未被软删除。
- 新后台路由必须在 `permissionForRoute` 中明确映射；未知后台资源默认要求 `system:read`，不默认放行。
- 403 拒绝不会执行业务 handler，并记录 `denied` 审计事件。
- Seed 可重复执行，始终为管理员补授 `super_admin`，并为超级管理员同步所有新增权限。
- 用户/角色管理 API 已实现最后一个有效超级管理员保护；停用和角色替换共用事务级 advisory lock，并发操作也不能绕过。
- 创建用户的明文密码只用于 bcrypt，不进入响应、审计 JSON 或错误消息。
